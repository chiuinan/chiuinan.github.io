簡介：日本Will公司2002年出品的18禁麻將遊戲，稱為人妻麻雀，2007年由未來數位中文化
      後更名並代理發行。遊戲採用情色劇情故事進行，描寫主角為尋找十年前的救命恩人，
      從琉球一路往北到札幌，其間與6位人妻打麻將做愛的故事，中途再結合日制13張麻將
      遊戲，並以其勝負做為劇情的分支。整個劇情頗為情色，圖片也還算不錯，還有日文
      語音搭配，因此娛樂性相當不錯。不過情色部份偏向於重口味的調教系，包括飲尿、
      浣腸、肛交等，某些玩家可能比較無法接受。至於麻將部份，並沒有刻意讓玩家容易
      進張與胡牌，完全靠玩家自身的運氣與牌技，又因為中途無法存取檔，加上電腦亦有
      作弊之嫌，因此要獲勝有相當的難度。此外，麻將必須打完半雀才能定勝負，玩家若
      想要全窺各種分支的劇情，需要花上不少時間。但整體而言，這款遊戲還是值得喜歡
      情色遊戲的玩家玩玩看的。

資訊：arc檔結構

檔案形態數目：4 byte
檔案形態資訊：
  形態：4 byte，ASCIIZ字串
        "SCR" = 顯示的相關資料（Rio.arc）
        "WIP" = 圖片檔（Chip.arc）
  檔案數：4 byte
  檔案資訊位置：4 byte
檔案資訊：每個17 byte
  檔名：9 byte = 8 byte名稱 + 1 byte NULL字元
  大小：4 byte
  位置：4 byte
檔案資料：
  "SCR"資料的解密法為各byte值右轉2位元(ror 2)
  其餘資料不加密

WIP圖像檔格式：

檔頭：32 byte
  標記：4 byte，固定"WIPF"
  不明：2 byte
  不明：2 byte
  水平大小：4 byte
  垂直大小：4 byte
  不明：12 byte
  壓縮資料大小：4 byte
  額外資料：大小不明，通常沒有
圖像壓縮資料，解壓後資料為：
  B資料：垂直大小*水平大小 byte
  G資料：垂直大小*水平大小 byte
  R資料：垂直大小*水平大小 byte

WIP圖像資料解壓法：

buffer = 4096 byte解壓緩衝區
bindex = 1
while (true)
{
  flag = 讀取1 byte
  由低位元開始檢測flag的8個位元值
    if 位元值 = 1 then
    {
      c = 讀取1 byte
      輸出c
      buffer[bindex] = c
      bindex = (bindex+1) % 4096
    }
    else
    {
      d = 讀取2 byte整數,高位元組在前
      重複位置 = d / 16
      if 重複位置 = 0 then 解壓結束返回(不足部份以最後一個輸出填滿)
      重複長度 = d % 16 + 2
      repeat 重複長度
      {
        c = buffer[重複位置]
        輸出c
        buffer[bindex] = c
        bindex = (bindex+1) % 4096
        重複位置 = (重複位置+1) % 4096
      }
    }
}

攻略：（作者—青衫）

1.麻將遊戲以半雀決勝負（東、南兩圈），中途若有人籌碼不足仍繼續進行。
2.採日式13張麻將規則計台（有番才能胡），同時有紅五寶牌。
3.劇情會依主角對女對手的排名差距進行分支，若敗於女對手，則可繼續挑戰。
4.前五次麻將的排名差總和必須超過8，否則第六位前會視為壞結局結束。
5.由於劇情分支的依據是排名差距，因此除了提高自己籌碼外，也要積極降低女對手的籌碼。
  同時務必取得排名差距2以上，才進入下個劇情。
6.由於日制13張麻將必須有番才能胡牌，因此在沒有切確的番牌前（如圈風刻、三元刻等），
  儘可能不要吃碰牌，以便利用立直做成一番胡牌。
7.日制13張麻將胡牌所能獲得的籌碼，隨著番數而呈倍數增長，因此要儘可能做大牌。
8.如果手上有多張寶牌（翻開牌的下一數字）、番數很多時，便要搶快聽牌、胡牌，此時便
  可以開始吃碰牌。但記得要做成有番的胡牌形式，最容易的便是捨棄所有的么九牌與字牌，
  以形成斷么九。
9.劇情模式有打過牌的人，才能在自由對戰中選擇。自由對戰可決定各種計番規則。

*** 麻將說明

1.食斷：以鳴牌（吃、碰、明槓）的方法作出只有斷么九的牌型。
2.一發：立直叫聽之後，在下次摸牌或摸牌之前就胡牌。
3.聽牌懸賞：如果有立直且胡牌的話，打開原懸賞牌下面的那張牌是否可以計算懸賞。
4.槓牌懸賞：如果有槓牌且胡牌的話，打開原懸賞牌右邊的那張牌是否可以計算懸賞。
5.紅牌懸賞：紅色五萬、五索、五筒的數量，如果持有並胡牌即可計算懸賞。
6.沒聽罰符全場3000點，形式聽牌當作聽牌：
  有一人聽牌時，沒聽的人給聽牌的人1000點。
  兩人聽牌時，沒聽的人給聽牌的人1500點。
  三人聽牌時，沒聽的人給所有聽牌的人各1000點。
  全部聽牌時，點數維持不變。
7.立直叫聽有一台，不是門清時不能立直，宣告立直需支付1000點給這局胡牌的人。
