整數大數運算演算法研究 -- 邱奕南，2009/3/16

所謂整數大數運算，指的是整數數目超過電腦暫存器可儲存的極限，必須切分成多個小整數來加以表
示。例如32位元電腦能運算的整數值最大必須小於4G，超過這個數目，便必須進行整數大數運算。由
於整數大數運算，多採用陣列結構，故亦常當作資料結構的一個練習題。

整數大數運算和多項式（Polynomial）運算相當相似，因此其運算演算法亦大多類似。而現今的整數
大數運算，則多運用在RSA加解密運算。RSA加解密運算的基礎為：

    y = x^e mod n

n稱為Modulus，一般要求在1024位元（128 byte）以上，因此便需要1024位元以上的整數大數乘法與
餘數運算。而這兩個運算的效率，便成為RSA加解密效率的關鍵。

在開始探討整數大數運算演算法前，我們需要先定出表示整數大數的結構。其結構有兩種表示方式，
一是以固定大小的陣列來表示，一是以動態記憶體的陣列來表示。兩者的優缺點為：

1.固定大小陣列：
  (1)只能運算限定位數以內的整數大數，超過便會Overflow（結果不正確）
  (2)遞迴運算時，可能導致堆疊Overflow而程式錯誤結束
  (3)運算時可能需要進行資料搬移而減低速度
  (4)採用operator運算時，可能會因大量資料拷貝而減低速度（視Compiler最佳化而異）

2.動態大小陣列：
  (1)可運算不定位數的整數大數
  (2)運算時，可能因記憶體不足而導致運算失敗
  (3)運算時，需要額外的動態記憶體配置與還回的時間，但可藉由指標轉移省去資料搬移時間
  (4)採用operator運算時，會因暫存物件拷貝與解構問題，導致動態記憶體被誤還回

兩者均各有其優缺點。以下我們先以固定大小陣列為準，最後再修正為動態記憶體陣列，以符合各種
的用途。而每個陣列元素（小整數）亦考慮1、2、4 byte等不同的大小，以供效率的比較。

接下來便是要考慮數目在陣列中的表示方式。RSA公私鑰均以ANS.1 DER編碼結構來呈現，其整數數值
是以位元組進行拆分，高位元組在前，低位元組在後。對於整數大數運算而言，常需要有運算基準線，
對於固定大小的陣列而言，容易以右邊靠齊，因左邊進位後超過陣列空間，即視為溢位錯誤。然而對
於動態大小的陣列而言，進位後超過陣列空間，表示必須重新配置記憶體空間，然後再搬移資料，這
點會影響到運算速度，故此處我們採用低位數在前的表示法。而為了避免記錄過多不必要的前導0，
所有的前導0均不加以記錄。依此，我們可寫出一些基本的處理函數（直接置入class BigNumber裡）：

（二）加法與減法

兩個n位元數的相加，其效率必定是O(n)，這點是無庸置疑的，所差別的，只是單純寫法上的效率差
異而已。加法中最需要考慮的便是進位的問題，在組合語言裡有add與adc的輔助，而直接將進位加入
數值中，但在C語言便沒有這麼方便了。一種通用性的判斷法，假設C=A+B，則：

    if (C < max(A,B)) then 有進位

這點是很容易證明的。當沒有進位時，C值必定>=A或B，有進位時，進位在C裡已經移出，於是形成相
加後變小的情況。當然，這樣的判斷法，速度會稍慢些。對於陣列元素的大小較CPU暫存器為小者，
其實也可利用多出的部份做進位記錄。例如陣列元素為byte者，便可利用short來加以記錄並判斷進
位：

    unsigned char A, B; // 要加的兩數字 //
    unsigned short D;   // 暫存用空間 //
    unsigned char C = (unsigned char) (D = A+B);
    unsigned char carry = *((char*)&D+1); // 假設資料在記憶體裡是低位元組在前 //

最後的carry實際上不必取出，而直接在下次要加時，直接以*((char*)&D+1)的形態加入數值中。
*((char*)&D+1)見似寫法複雜，對組語而言只是一個位址，且會與前後的運算子，如=、+等合併一起，
本身並不會耗時。然而這樣的寫法會隨著CPU的存取格式（Big Endian，即高位元組在前，Little 
Endian，即低位元組在前）與陣列元素大小而異。由於此部份影響效率的程度並不高，故此處我們仍
採用通用性判斷法。以下為加入一位數的函數：

void BigNumber::Add(const BigNumber& bn)
//說明：加入一個數值
//輸入：bn = 欲加入的數值
{
    if (bn.Is0()) return; // 0不用加 //
    if (Is0()) // 本身為0，直接複製 //
    {
        *this = bn;
        return;
    }
    if (IsNeg())
    {
        if (bn.IsNeg()) // 兩數皆負，直接相加 //
        {
            add_abs(bn);
        }
        else
        {
            // A負B正 //
            int result = compare_abs(bn);
            if (result < 0) // A+B變正, 計算|B|-|A| //
            {
                BigNumber tbn;
                tbn = bn;
                Abs();
                tbn -= *this;
                *this = tbn;
            }
            else if (result > 0) // A+B仍負, 計算|A|-|B| //
            {
                sub_abs(bn);
            }
            else // A=B, 相加變0 //
            {
                Set0();
            }
        }
    }
    else
    {
        if (bn.IsNeg())
        {
            // A正B負 //
            int result = compare_abs(bn);
            if (result < 0) // A+B變負, 計算|B|-|A| //
            {
                BigNumber tbn;
                tbn = bn;
                tbn.Abs();
                tbn -= *this;
                *this = tbn;
                Neg();
            }
            else if (result > 0) // A+B仍正, 計算|A|-|B| //
            {
                sub_abs(bn);
            }
            else // A=B, 相加變0 //
            {
                Set0();
            }
        }
        else // 兩數皆正，直接相加 //
        {
            add_abs(bn);
        }
    }
}

（三）乘法

兩個整數大數相乘，最簡單的做法便是採用傳統的移位累加法，亦即：

             7 3
     *       2 9
    -------------
             2 7 (9*3)
           6 3   (9*7)
           0 6   (2*3)
     +   1 4     (2*7)
    -------------
         2 1 1 7

對於兩個n位數相乘，需要n^2個基礎乘法，故其效率為O(n^2)。傳統移位累加法在撰寫時有兩種方式，
一是每次乘得兩位數時，即分別加入結果中，一是按乘數的每位數計算後，集結成兩個數字陣列，再
進行兩個整數大數加法。例如前例以9乘時，可得{3,7}、{6,2}兩個陣列，以2乘時，可得{4,6}、{1,0}
兩個陣列，各陣列依其基準位加入結果數中。而為了比較採用組合語言，以及不同陣列元素大小的執
行效率，以下的函數均按不同的定義進行不同的寫法：

        return;
    }

（四）除法

兩個整數大數相除，最簡單的做法便是採用傳統的移位累減法，亦即：

              2 5 ------> 商
       +-------------
  2 9  |    7 3 0
         -) 5 8
        -------------
            1 5 0
         -) 1 4 5
        -------------
                5 ------> 餘數

對於2n位數除以n位數的計算，共需n^2次乘法，故這個演算法的效率一樣是O(n^2)。這個演算法另一
個效率的限制，則在於如何預測出可能的位數商值。在電腦運算上，多以位元為單位進行運算，因此
位數商值不是0便是1，比較沒有預測的問題，然而這類的運算需要移位運算做為輔助。對於整數大數
而言，移位也是個相當緩慢的運算，OpenSSL的source code裡已明白指出這種計算法速度很慢，故此
處我們採用以陣列元素單位做為運算的基準，這也是OpenSSL裡的做法。由於位數商值多以除數首位
數進行預測，故我們先導出其預測誤差模式以供判斷。

假設陣列元素單位值為t（bit=2，byte=256，2 byte=65536等），而目前納入預測的被除數的前3位
數為a、b、c，除數前2位數為d、e，其中：

    0 <= a <= d
    1 <= d <= t
    0 <= b,c,e <= t

一般預測的位數商值多以(a*t+b)/d為準，故：

    (a*t+b)/d = f，餘數g，0<=g=t/2，也就是最高位元為1，這點可在預測位數商值時，
進行移位運算，由於只處理最多5個位數，因此並不會太耗時。如此最差的情況為d=t/2，f=t-1，
e=t-1時，可算得x約為2～3，變成一個相當理想的誤差值。

事實上，如何準確預測位數商值，有許多的文件述及，甚至可將誤差值限縮到1以內，但計算偏於複
雜，實際能提升的效能很有限，故無需對此一部份過份精研。

int BigNumber::div_abs_basic(BIG_NUMBER_DATA_TYPE* data1, int len1, BIG_NUMBER_DATA_TYPE dvalue, BIG_NUMBER_DATA_TYPE* result, int &ret_result_len)

//輸入：data1, len1 = 被除數, dvalue = 除數, >1

//備註：本函數採用傳統移位除法，效率為O(n^2)
{
    int remain_len = 0;
    BIG_NUMBER_DATA_TYPE qvalue, hvalue, lvalue;
    for (int i=len1-1; i>=0; --i)
    {
        if (remain_len == 0)
        {
            if ((lvalue=data1[i]) == 0) // 被除數為0 //
            {
                result[i] = 0;
            }
            if (lvalue < dvalue)
            {
                // 被除數小於除數，結果為0，餘數不變 //
                result[i] = 0;
                ++remain_len;
            }
            else if (dvalue == lvalue)
            {
                // 被除數等於除數，結果為1，餘數為0 //
                result[i] = 1;
            }
            else
            {
                result[i] = qvalue = lvalue/dvalue;
                if ((data1[i] = lvalue % dvalue) != 0) // 非整除, 餘數留到下回 //
                {
                    ++remain_len;
                }
            }
            continue;

        hvalue = data1[i+1];

            mov ah, byte ptr [hvalue]

#elif BIG_NUMBER_DATA_TYPE_SIZE == 2

#endif
        qvalue = (BIG_NUMBER_DATA_TYPE)(value / dvalue);

        result[i] = qvalue;
        if ((data1[i] = lvalue) == 0) remain_len = 0;

//輸出：data1 = 餘數, result, ret_result_len = 商 (未標準化)
//傳回：餘數長度 (未標準化)
//備註：本函數採用傳統移位除法，效率為O(n^2)
{

    {
        divide_by_zero();
        ret_result_len = 0;
        return 0;
    }
    if (len1 < len2) // 被除數較小, C=1,R=A (被除數0時同, C=0,R=0) //
    {
        ret_result_len = 0;
        return len1;

    if (len2 == 1)
    {

            ret_result_len = len1;
            return 0;

        else
        {
            return div_abs_basic(data1,len1,data2[0],result,ret_result_len);
        }

    int remain_len = len2-1;

                mov al, byte ptr [lvalue]
                mov ah, byte ptr [hvalue]
                div byte ptr [dvalue]
                mov byte ptr [qvalue],al
            }
#elif BIG_NUMBER_DATA_TYPE_SIZE == 2
            __asm
            {
                mov ax, word ptr [lvalue]
                mov dx, word ptr [hvalue]
                div word ptr [dvalue]
                mov word ptr [qvalue],ax
            }
#elif BIG_NUMBER_DATA_TYPE_SIZE == 4
            __asm
            {
                mov eax, dword ptr [lvalue]
                mov edx, dword ptr [hvalue]
                div dword ptr [dvalue]
                mov dword ptr [qvalue],eax
            }
#else
    #error Not Implement
#endif
#else
#if (BIG_NUMBER_DATA_TYPE_SIZE == 1) || (BIG_NUMBER_DATA_TYPE_SIZE == 2)

bool BigNumber::mul_abs_divide(const BIG_NUMBER_DATA_TYPE* data1, int len1, const BIG_NUMBER_DATA_TYPE* data2, int len2, BIG_NUMBER_DATA_TYPE* data3, int &len3)
//說明：數值相乘（絕對值）
//輸入：data1, len1, data2, len2 = 欲相乘的數值
//輸出：data3, len3 = 相乘後的數值 (未標準化)
//傳回：記憶體不足傳回false
//限制：data3必須有足夠空間
//備註：本函數採用分割解決法（最大分割），亦即x*y = (a*2^n+b)(c*2^n+d)，則
//          u = (a+b)*(c+d)
//          v = a*c
//          w = b*d
//          z = v*2^2n + (u-v-w)*2^n + w
//      效率為O(n^log3)=O(n^1.59)
{
    len3 = 0;
    if ((len1 <= MUL_DIVIDE_BOUND) || (len2 <= MUL_DIVIDE_BOUND))
    {
        if ((len1 == 0) || (len2 == 0)) return true;
        len3 = mul_abs_basic(data1,len1,data2,len2,data3);
        return true;
    }
    int len = max(len1,len2);
    // 配置暫存空間 //
    BIG_NUMBER_DATA_TYPE *udata, *vdata;
    int need_no = len*2;
    if ((udata = alloc_data(need_no)) == NULL) return out_of_memory();
    if ((vdata = alloc_data(need_no)) == NULL) 
    {
        delete[] udata;
        return out_of_memory();
    }
    // 決定切分後各部份的長度 //
    len /= 2;
    const BIG_NUMBER_DATA_TYPE *v, *w;
    int alen, blen, clen, dlen, ulen, vlen, wlen, rlen3;
    if (len < len1)
    {
        alen = len1-len;
        blen = ::standardize(data1,len);
    }
    else 
    {
        alen = 0;
        blen = len1;
    }
    if (len < len2) 
    {
        clen = len2-len;
        dlen = ::standardize(data2,len);
    }
    else 
    {
        clen = 0;
        dlen = len2;
    }
    // 算出u //
    // a+b //
    if (alen == 0) // a=0, 直接使用b //
    {
        w = data1;
        wlen = blen;
    }
    else if (blen == 0) // b=0, 直接使用a //
    {
        w = data1+len;
        wlen = alen;
    }
    else // 以data3為空間, 計算w=a+b //
    {
        wlen = add_abs(data1+len,len1-len,data1,blen,data3);
        w = data3;
    }
    // c+d //
    if (clen == 0) // c=0, 直接使用d //
    {
        v = data2;
        vlen = dlen;
    }
    else if (dlen == 0) // d=0, 直接使用c //
    {
        v = data2+len;
        vlen = clen;
    }
    else // 以vdata為空間, 計算v=c+d //
    {
        vlen = add_abs(data2+len,len2-len,data2,dlen,vdata);
        v = vdata;
    }
    if (!mul_abs_divide(w,wlen,v,vlen,udata,ulen)) // 使用udata空間, u=w*v=(a+b)*(c+d) //
    {
        delete[] vdata;
        delete[] udata;
        return false;
    }
    // 算出v //
    if ((alen == 0) || (clen == 0)) // a或c=0, v=a*c=0 //
    {
        vlen = 0;
    }
    else 
    {
        if (!mul_abs_divide(data1+len,alen,data2+len,clen,vdata,vlen)) // 使用vdata空間, v=a*c //
        {
            delete[] vdata;
            delete[] udata;
            return false;
        }
    }
    // 算出w //
    if ((blen == 0) || (dlen == 0)) // b或d=0, w=b*d=0 //
    {
        rlen3 = 0;
    }
    else
    {
        if (!mul_abs_divide(data1,blen,data2,dlen,data3,rlen3)) // 使用data3空間, data3=b*d //
        {
            delete[] vdata;
            delete[] udata;
            return false;
        }
    }
    // 計算u-v-w //
    sub_abs(udata,ulen,vdata,vlen); // u-=v //
    sub_abs(udata,ulen,data3,rlen3); // u-=w //
    ulen = ::standardize(udata,ulen);
    // 進行移位相加 //
    while (rlen3 < len) data3[rlen3++] = 0;
    rlen3 = add_abs(data3+len,rlen3-len,udata,ulen,data3+len) + len;
    if (vlen != 0)
    {
        len *= 2;
        while (rlen3 < len) data3[rlen3++] = 0;
        rlen3 = add_abs(data3+len,rlen3-len,vdata,vlen,data3+len) + len;
    }
    delete[] vdata;
    delete[] udata;
    len3 = rlen3;
    return true;
}

//說明：數值相乘（絕對值）

bool BigNumber::mul_abs(const BigNumber& bn)
//說明：數值相乘（絕對值）
//輸入：bn = 乘數
//傳回：記憶體不足傳回false
{
    bool call_divide = true;
    if ((m_Len <= MUL_DIVIDE_BOUND) || (bn.GetLen() <= MUL_DIVIDE_BOUND)) 
    {
        call_divide = false;
    }
    else
    {
        // 偵測非0值是否有超過MUL_DIVIDE_BOUND //
        const BIG_NUMBER_DATA_TYPE* data = m_Data;
        int nonzero_count = 0;
        for (int i=m_Len-1; i>=0; i--)
        {
            if (data[i] == 0) continue;
            if (++nonzero_count > MUL_DIVIDE_BOUND) break;
        }
        if (i < 0)
        {
            call_divide = false;
        }
        else
        {
            data = bn.GetData();
            nonzero_count = 0;
            for (int i=bn.GetLen()-1; i>=0; i--)
            {
                if (data[i] == 0) continue;
                if (++nonzero_count > MUL_DIVIDE_BOUND) break;
            }
            if (i < 0) call_divide = false;
        }
    }
    if (call_divide) return mul_abs_divide(bn);
    else return mul_abs_basic(bn);
}

int BigNumber::div_abs_basic(BIG_NUMBER_DATA_TYPE* data1, int len1, BIG_NUMBER_DATA_TYPE dvalue, BIG_NUMBER_DATA_TYPE* result, int &ret_result_len)
//說明：數值相除（絕對值）
//輸入：data1, len1 = 被除數, dvalue = 除數, >1
//輸出：data1 = 餘數, result, ret_result_len = 商 (未標準化)
//傳回：餘數長度 (未標準化)
//限制：result必須有足夠空間
//備註：本函數採用傳統移位除法
{
    int remain_len = 0;
    BIG_NUMBER_DATA_TYPE qvalue, hvalue, lvalue;
    for (int i=len1-1; i>=0; --i)
    {
        if (remain_len == 0)
        {
            if ((lvalue=data1[i]) == 0) // 被除數為0 //
            {
                result[i] = 0;
            }
            if (lvalue < dvalue)
            {
                // 被除數小於除數，結果為0，餘數不變 //
                result[i] = 0;
                ++remain_len;
            }
            else if (dvalue == lvalue)
            {
                // 被除數等於除數，結果為1，餘數為0 //
                result[i] = 1;
            }
            else
            {
                result[i] = qvalue = lvalue/dvalue;
                if ((data1[i] = lvalue % dvalue) != 0) // 非整除, 餘數留到下回 //
                {
                    ++remain_len;
                }
            }
            continue;
        }
        // 2位數除法 //
        lvalue = data1[i];
        hvalue = data1[i+1];
        // qvalue = hvalue:lvalue/dvalue //
#if (BIG_NUMBER_DATA_TYPE_SIZE == 1) || (BIG_NUMBER_DATA_TYPE_SIZE == 2)
        unsigned int value = ((unsigned int)hvalue << (BIG_NUMBER_DATA_TYPE_SIZE*8)) + lvalue;
        qvalue = (BIG_NUMBER_DATA_TYPE)(value / dvalue);
        lvalue = (BIG_NUMBER_DATA_TYPE)(value % dvalue);
#elif BIG_NUMBER_DATA_TYPE_SIZE == 4
#ifdef USE_ASM
        __asm
        {
            mov eax, dword ptr [lvalue]
            mov edx, dword ptr [hvalue]
            div dword ptr [dvalue]
            mov dword ptr [qvalue],eax
            mov dword ptr [lvalue],edx
        }
#else
        unsigned __int64 value = ((unsigned __int64)hvalue << (BIG_NUMBER_DATA_TYPE_SIZE*8)) + lvalue;
        qvalue = (BIG_NUMBER_DATA_TYPE)(value / dvalue);
        lvalue = (BIG_NUMBER_DATA_TYPE)(value % dvalue);
#endif
#else
    #error Not Implement
#endif
        result[i] = qvalue;
        if ((data1[i] = lvalue) == 0) remain_len = 0;
    }
    ret_result_len = len1;
    return remain_len;
}

bool BigNumber::div_abs_basic(BIG_NUMBER_DATA_TYPE* data1, int len1, int &ret_len1, const BIG_NUMBER_DATA_TYPE* data2, int len2, BIG_NUMBER_DATA_TYPE* result, int &ret_result_len)
//說明：數值相除（絕對值）
//輸入：data1, len1 = 被除數, data2, len2 = 除數, >1, 不與被除數佔用相同空間
//輸出：data1, ret_len1 = 餘數, result, ret_result_len = 商 (未標準化)
//傳回：記憶體不足傳回false
//限制：result必須有足夠空間
//備註：本函數採用傳統移位除法
{
    ret_len1 = len1;
    ret_result_len = 0;
    if (len1 < len2) return true; // 被除數較小, C=1,R=A (被除數0時同, C=0,R=0) //
    if (len2 == 1)
    {
        ret_len1 = div_abs_basic(data1,len1,data2[0],result,ret_result_len);
        return true;
    }
    // 開始移位除法, 以下除數最少2位以上 //
    int remain_len = len2-1;
    int compare_result, temp_len;
    BIG_NUMBER_DATA_TYPE dvalue = data2[len2-1];
    BIG_NUMBER_DATA_TYPE dvalue2 = data2[len2-2];
    BIG_NUMBER_DATA_TYPE hbit = 1 << (sizeof(BIG_NUMBER_DATA_TYPE)*8-1);
    // 將試除數轉成最高bit為1, 以減少商值猜測時之誤差 (目前誤差最多約為3) //
    int shift_bit_no = 0;
    while ((dvalue & hbit) == 0)
    {
        dvalue <<= 1;
        if (dvalue2 & hbit) dvalue |= 1;
        dvalue2 <<= 1;
        ++shift_bit_no;
    }
    BIG_NUMBER_DATA_TYPE qvalue, hvalue, lvalue, nvalue;
    BIG_NUMBER_DATA_TYPE *temp;
    if ((temp = alloc_data(len1)) == NULL) return false;
    for (int i=len1-len2; i>=0; --i)
    {
        if ((remain_len++ == 0) && (data1[i] == 0)) // 被除數為0 //
        {
            result[i] = remain_len = 0;
            continue;
        }
        if ((compare_result = compare_abs(data1+i,remain_len,data2,len2)) < 0)
        {
            // 被除數小於除數，結果為0，餘數不變 //
            result[i] = 0;
            continue;
        }
        if (compare_result == 0)
        {
            // 被除數等於除數，結果為1，餘數為0 //
            result[i] = 1;
            remain_len = 0;
            continue;
        }
        // 猜測商可能的值 //
        if (len2 < remain_len)
        {
            hvalue = data1[i+remain_len-1];
            lvalue = data1[i+remain_len-2];
            nvalue = data1[i+remain_len-3];
        }
        else
        {
            hvalue = 0;
            lvalue = data1[i+remain_len-1];
            nvalue = data1[i+remain_len-2];
        }
        // 進行與試除數相同移位 //
        if (shift_bit_no != 0)
        {
            hvalue = (hvalue << shift_bit_no) | (lvalue >> (sizeof(BIG_NUMBER_DATA_TYPE)*8-shift_bit_no));
            lvalue = (lvalue << shift_bit_no) | (nvalue >> (sizeof(BIG_NUMBER_DATA_TYPE)*8-shift_bit_no));
        }
        if (hvalue >= dvalue)
        {
            // 位數不同，若餘數最高位數值 >= 除數最高位數值，必為最大值 //
            qvalue = (BIG_NUMBER_DATA_TYPE)-1;
        }
        else
        {
            // 找出可能的商 = hvalue:lvalue/dvalue //
#if (BIG_NUMBER_DATA_TYPE_SIZE == 1) || (BIG_NUMBER_DATA_TYPE_SIZE == 2)
            qvalue = (((unsigned int)hvalue << (BIG_NUMBER_DATA_TYPE_SIZE*8)) + lvalue) / dvalue;
#elif BIG_NUMBER_DATA_TYPE_SIZE == 4
            qvalue = (BIG_NUMBER_DATA_TYPE)((((unsigned __int64)hvalue << (BIG_NUMBER_DATA_TYPE_SIZE*8)) + lvalue) / dvalue);
#else
    #error Not Implement
#endif
        }
        temp_len = mul_abs_basic(data2,len2,&qvalue,1,temp);
        temp_len = ::standardize(temp,temp_len);
        // 檢查實際結果修正商 (誤差愈少, 速度愈快) //
        while ((compare_result = compare_abs(data1+i,remain_len,temp,temp_len)) < 0)
        {
            // 結果過大，商減1 //
            sub_abs(temp,temp_len,data2,len2);
            temp_len = ::standardize(temp,temp_len);
            --qvalue;
        }
        if (compare_result == 0)
        {
            // 相等，餘數為0 //
            result[i] = qvalue;
            remain_len = 0;
            continue;
        }
        result[i] = qvalue;
        sub_abs(data1+i,remain_len,temp,temp_len);
        remain_len = ::standardize(data1+i,remain_len);
    }
    ret_len1 = remain_len;
    ret_result_len = len1-len2+1;
    delete[] temp;
    return true;
}

無論乘法或除法，兩種方法的效率都差別不大，但大抵動態大小陣列稍好一些。動態大小陣列效率的
增加，主要是來自於計算後的資料無需拷回，但卻因需動態記憶體配置與釋放而抵消不少。特別是乘
法中的分割解決法，對於兩個n位數相乘，約需n/8次動態記憶體配置，耗去不少時間，但也因此避免
了堆疊滿溢的問題。


三．其他的乘除法演算法

乘法演算法尚有Toom-Cook的多重分割解決法，效率為O(n^(1+3.5/sqrt(logn)))；DFT法，效率為
O(nlognloglogn)。

除法演算法尚有Fourier Division，效率為O(n^2)；Reciprocal法（牛頓逼近法），效率同乘法演算
法。

這些演算法皆留待將來再研究。其他諸如移位等運算，亦留待將來再補充。
